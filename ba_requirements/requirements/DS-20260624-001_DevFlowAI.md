## 1. 概述

### 1.1 项目名称
DevFlow AI - 智能开发工作流助手

### 1.2 功能简述
一套完整的 **FS → DS → Code → Review** 开发工作流系统，通过 DS 文档作为 AI Coding Agent 的唯一需求输入，实现需求精确传达、开发边界清晰定义、代码与需求可追溯对照。

### 1.3 关联 FS
- 文档：FS-20260624-001_DevFlowAI.md
- 核心功能：
  a. 📄 文档管理中心（FS/DS 上传、解析、管理）
  b. 🤖 AI Coding Agent 工作台
  c. 📊 工作流可视化（优先实现）
  d. 🔍 进阶 Code Review

### 2.3 数据模型

#### 2.3.1 项目 (Project)
```python
class Project(BaseModel):
    id: str              # UUID
    name: str            # 项目名称
    description: str     # 项目描述
    created_at: datetime # 创建时间
    updated_at: datetime # 更新时间
    fs_document_id: str  # 关联 FS 文档 ID
```

#### 2.3.2 文档 (Document)
```python
class Document(BaseModel):
    id: str              # UUID
    project_id: str      # 所属项目
    type: str            # "fs" | "ds"
    name: str            # 文档名称
    content: str         # 文档内容 (Markdown)
    version: int         # 版本号
    created_at: datetime
    updated_at: datetime
    status: str          # "draft" | "published"
```

#### 2.3.3 Agent 配置 (AgentConfig)
```python
class AgentConfig(BaseModel):
    id: str
    project_id: str
    model_provider: str  # "openai" | "anthropic" | "google"
    model_name: str
    temperature: float   # 0.0-1.0
    max_tokens: int
    system_prompt: str
    tools: list[str]     # 可用工具列表
```

#### 2.3.4 执行任务 (ExecutionTask)
```python
class ExecutionTask(BaseModel):
    id: str
    project_id: str
    ds_document_id: str
    status: str          # "pending" | "running" | "paused" | "completed" | "failed"
    current_step: int
    steps: list[Step]
    created_at: datetime
    started_at: datetime
    completed_at: datetime
```

#### 2.3.5 执行步骤 (Step)
```python
class Step(BaseModel):
    id: int
    description: str     # DS 中的步骤描述
    acceptance: str      # 验收条件
    status: str          # "pending" | "in_progress" | "completed" | "failed"
    tool_calls: list[ToolCall]
    result: str          # 执行结果
    error: str           # 错误信息（如有）
    started_at: datetime
    completed_at: datetime
```

#### 2.3.6 Tool 调用记录 (ToolCall)
```python
class ToolCall(BaseModel):
    step_id: int
    tool_name: str
    input_params: dict
    output: str
    duration_ms: int
    status: str          # "success" | "failed"
    timestamp: datetime
```

## 3. 接口设计

### 3.1 文档管理 API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/v1/projects | 创建项目 |
| GET | /api/v1/projects | 获取项目列表 |
| GET | /api/v1/projects/{id} | 获取项目详情 |
| POST | /api/v1/projects/{id}/documents | 上传文档 |
| GET | /api/v1/projects/{id}/documents | 获取文档列表 |
| GET | /api/v1/documents/{id} | 获取文档详情 |
| PUT | /api/v1/documents/{id} | 更新文档 |
| POST | /api/v1/documents/{id}/versions | 创建版本快照 |
| GET | /api/v1/documents/{id}/versions | 获取版本历史 |
| POST | /api/v1/documents/{id}/rollback/{version} | 回滚版本 |

### 3.2 Agent API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/v1/agent/config | 创建 Agent 配置 |
| GET | /api/v1/agent/config/{project_id} | 获取 Agent 配置 |
| PUT | /api/v1/agent/config/{id} | 更新 Agent 配置 |
| POST | /api/v1/agent/execute | 开始执行任务 |
| GET | /api/v1/agent/execute/{task_id} | 获取任务状态 |
| POST | /api/v1/agent/execute/{task_id}/pause | 暂停任务 |
| POST | /api/v1/agent/execute/{task_id}/resume | 继续任务 |
| POST | /api/v1/agent/execute/{task_id}/step/{step_id}/approve | 人工确认 |
| GET | /api/v1/agent/execute/{task_id}/logs | 获取执行日志 |

### 3.3 Code Review API

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/v1/review/analyze | 分析代码与 DS 对比 |
| GET | /api/v1/review/{task_id} | 获取 Review 结果 |
| GET | /api/v1/review/{task_id}/report | 生成 Review 报告 |
