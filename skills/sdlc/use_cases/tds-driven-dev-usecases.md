# 用例 - 技术驱动开发技能

## 用例1：新功能实现

### 场景
开发者收到已批准的TDS，需要实现"用户认证"功能

### 输入
- TDS文档: `requirements/TDS-20260624-001_DevFlowAI.md`
- 章节: 3.1 用户认证

### 流程
1. 解析TDS获取实现步骤
2. 初始化项目结构
3. 实现数据模型
4. 创建API端点
5. 添加业务逻辑
6. 编写单元测试

### 输出
- 带REQ-TDS-XXX注释的代码
- 单元测试
- 带需求引用的提交历史

### 验证
- [ ] 所有代码有需求注释
- [ ] 代码编译无错误
- [ ] 单元测试通过

## 用例2：带TDS可追溯性的Bug修复

### 场景
QA报告了项目创建中的一个bug - 描述字段未保存

### 输入
- Bug报告: `issues/bug-001.md`
- 原始TDS: `requirements/TDS-20260624-001_DevFlowAI.md`

### 流程
1. 识别相关TDS需求(REQ-TDS-003)
2. 创建修复分支
3. 实现修复
4. 为bug添加测试用例
5. 在提交中引用需求

### 输出
- 修复后的代码
- 更新的测试
- 提交: "20250627 - REQ-TDS-003 修复项目描述保存"

## 用例3：多团队功能实现

### 场景
需要前端和后端团队协调的大型功能

### 输入
- TDS: `requirements/TDS-20260624-003_MultiTeam.md`
- 团队分配已定义

### 流程
1. 按团队拆分TDS
2. 每个团队遵循tds_driven_dev技能
3. 定期同步会议
4. 集成测试

### 输出
- 协调的实现
- 集成测试
- 联合审查
