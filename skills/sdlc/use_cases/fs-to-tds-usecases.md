# 用例 - 需求转技术设计技能

## 用例1：新项目初始化

### 场景
BA团队提供了"DevFlow AI - 文档管理系统"的新FS文档

### 输入
- FS文档: `requirements/FS-20260624-002_DocMgmt.md`

### 流程
1. 需求分析师加载FS文档
2. 技能提取所有功能需求
3. 识别技术栈需求
4. 生成带实现步骤的TDS

### 输出
- TDS文档: `requirements/TDS-20260624-002_DocMgmt.md`

### 验证
- [ ] 所有FS功能已映射到TDS章节
- [ ] 每个功能有实现步骤
- [ ] 技术栈已验证

---

## 用例2：现有项目功能追加

### 场景
利益相关方要求在现有项目中添加"AI驱动的文档摘要"功能

### 输入
- 现有FS: `requirements/FS-20260624-001_DevFlowAI.md`
- 功能请求: 添加摘要功能

### 流程
1. 加载现有FS和TDS
2. 解析新功能需求
3. 添加到TDS作为新章节
4. 更新时间线

### 输出
- 更新的TDS，包含新功能章节

---

## 用例3：遗留系统迁移

### 场景
从遗留系统迁移，需要从现有文档创建TDS

### 输入
- 遗留FS(非正式): 各种文档和邮件

### 流程
1. 从各种来源汇总所有需求
2. 规范化为FS格式
3. 运行fs_to_tds技能生成TDS

### 输出
- 标准化的TDS文档
