## 技能定义

| 技能名称 | 用途 | 输入 | 输出 |
|----------|------|------|------|
| **fs_to_tds** | 功能规格转技术设计规格 | FS Markdown | TDS Markdown |
| **tds_driven_dev** | 技术设计驱动开发 | TDS Markdown | 代码+测试 |
| **code_review_tds** | 基于TDS的代码审查 | 代码+TDS | 审查报告 |
| **api_doc_gen** | 自动生成API文档 | 代码(API) | API文档(Markdown/OpenAPI) |
| **api_auto_regression** | API自动化回归测试 | API规范+测试 | 测试报告 |
| **bau_sop_gen** | 生成BAU标准操作程序 | 流程文档 | SOP Markdown |

## 角色定义

| 角色 | 视角 | 职责 |
|------|------|------|
| **需求分析师** | 需求分析 | 创建功能规格，与利益相关方确认 |
| **技能架构师** | 技术设计 | 设计技能、工作流、集成点 |
| **IT质量评审** | 质量保证 | 结果评审、测试验证 |
| **领导层** | 业务视角 | 查看报告、做出决策 |

## 技能链依赖关系

```
FS → [fs_to_tds] → TDS → [tds_driven_dev] → 代码
                                    ↓
                              [code_review_tds] → 审查报告
                                    ↓
                              [api_doc_gen] → API文档
                                    ↓
                              [api_auto_regression] → 测试结果
                                    ↓
                              [bau_sop_gen] → SOP文档
```

### GitHub Copilot集成
- 应用场景: `tds_driven_dev`, `code_review_tds`, `api_doc_gen`
- 收益: 代码补全、文档生成、测试生成
