# 用例 - 基于TDS的代码审查技能

## 用例1：合并前代码审查

### 场景
开发者提交了项目创建API的拉取请求

### 输入
- PR包含代码更改
- TDS: `requirements/TDS-20260624-001_DevFlowAI.md`

### 流程
1. 代码上传到审查队列
2. 技能扫描代码中的REQ-TDS-XXX注释
3. 验证实现是否符合TDS
4. 运行静态分析
5. 生成审查报告

### 输出
- 审查报告包含:
  - TDS合规性评分
  - 代码质量问题
  - 缺失实现
  - 操作项

### 验证
- [ ] 所有需求已处理
- [ ] 代码质量通过阈值
- [ ] IT质量评审已安排

## 用例2：IT质量评审结果审查

### 场景
IT质量团队需要验证实现完整性

### 输入
- 完成的实现
- 代码审查报告
- 测试结果

### 流程
1. 技能准备IT质量评审包
2. 运行最终验证检查
3. 生成IT质量评审结果报告
4. 创建签署文档

### 输出
- IT质量评审签署文档
- 部署批准

---

## 用例3：技术债务评估

### 场景
季度代码质量和技术债务审查

### 输入
- 仓库中的所有代码
- 历史审查报告

### 流程
1. 汇总所有REQ-TDS-XXX引用
2. 识别过时或陈旧的代码
3. 生成技术债务报告
4. 建议重构优先级

### 输出
- 技术债务报告
- 重构建议
