# 用例 - API文档生成技能

## 用例1：新API文档

### 场景
后端团队完成项目端点API实现

### 输入
- 包含API端点的源代码
- TDS: API规范

### 流程
1. 扫描源代码中的API端点
2. 从代码提取端点详细信息
3. 生成OpenAPI规范
4. 创建Markdown文档
5. 生成SDK

### 输出
- `docs/api/openapi.yaml`
- `wiki/api/projects.md`
- SDK包

### 验证
- [ ] 所有端点已文档化
- [ ] 已提供示例
- [ ] 可用于Wiki

## 用例2：API版本更新

### 场景
API v1需要更新到v2，包含破坏性更改

### 输入
- 当前API代码
- OpenAPI v1规范

### 流程
1. 识别已更改的端点
2. 生成差异报告
3. 创建迁移指南
4. 更新文档

### 输出
- 更新的OpenAPI规范
- 迁移指南
- 弃用通知

## 用例3：外部API消费者入职

### 场景
第三方开发者需要集成DevFlow AI API

### 输入
- 当前API文档

### 流程
1. 生成综合API文档
2. 创建Postman集合
3. 生成代码示例
4. 构建交互式示例

### 输出
- 完整的API文档包
- Postman集合
- 多语言代码示例
