# 用例 - API自动化回归测试技能

## 用例1：部署前验证

### 场景
开发者完成功能开发，想要部署到预发布环境

### 输入
- OpenAPI规范: `docs/api/openapi.yaml`
- TDS: 验收标准
- 测试数据: `tests/fixtures/`

### 流程
1. 从规范生成测试用例
2. 执行完整回归测试套件
3. 生成测试报告
4. 检查TDS验收标准

### 输出
- 测试报告: `results/regression_report.html`
- JUnit XML: `results.xml`
- 部署就绪状态

### 验证
- [ ] 所有测试通过
- [ ] TDS验收标准满足
- [ ] 性能基准达标

## 用例2：CI/CD集成

### 场景
作为拉取请求流水线的一部分进行自动化测试

### 输入
- PR中的代码更改
- OpenAPI规范
- 测试配置

### 流程
1. 在PR创建/更新时触发
2. 启动测试环境
3. 运行回归测试
4. 向PR报告结果
5. 如果测试失败则阻止合并

### 输出
- PR上的测试结果评论
- CI状态检查
- 详细报告产物

---

## 用例3：夜间构建验证

### 场景
计划的夜间测试以发现集成问题

### 输入
- 主分支的最新代码
- 完整测试套件
- 监控配置

### 流程
1. 拉取最新代码
2. 启动全新测试环境
3. 运行完整回归测试套件
4. 生成夜间报告
5. 失败时告警

### 输出
- 夜间测试报告
- 性能趋势
- 失败告警
