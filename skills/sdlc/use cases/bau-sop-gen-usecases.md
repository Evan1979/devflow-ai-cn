# 用例 - BAU运维手册生成技能

## 用例1：上线前运维手册生成

### 场景
系统下周上线，需要准备好所有运维SOP

### 输入
- API文档: `wiki/api/`
- 测试结果: `results/`
- 部署文档: `docs/deployment/`

### 流程
1. 收集所有源材料
2. 识别BAU活动
3. 生成监控SOP
4. 生成运维SOP
5. 生成故障响应SOP
6. 生成支持SOP

### 输出
- 完整SOP库: `wiki/sops/`
- 带导航的索引页
- 版本历史

### 验证
- [ ] 所有关键操作已覆盖
- [ ] 步骤已测试并可用
- [ ] 升级联系人是最新的

---

## 用例2：故障响应增强

### 场景
故障后，需要根据经验教训改进SOP

### 输入
- 故障报告: `incidents/20260620-api-down.md`
- 当前SOP: `wiki/sops/`

### 流程
1. 分析故障时间线
2. 识别当前SOP中的差距
3. 生成改进的程序
4. 添加故障排除步骤
5. 更新版本历史

### 输出
- 包含故障经验教训的更新SOP
- 新的故障排除章节
- 改进的恢复时间

---

## 用例3：新团队成员入职

### 场景
新运维团队成员需要综合文档

### 输入
- 所有生成的SOP
- 系统架构文档
- 联系人列表

### 流程
1. 生成入职指南
2. 编译快速参考
3. 创建FAQ文档
4. 构建故障排除索引
5. 准备知识检查

### 输出
- 入职指南: `wiki/onboarding/`
- 快速参考卡
- FAQ文档
